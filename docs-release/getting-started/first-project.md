# First Project Onboarding Guide

This guide details the procedure for establishing your first project within the dashboard.

## Prerequisites

- You must have successfully installed and started the Sprint OS local server.
- The dashboard must be running and accessible (default is `http://localhost:4444`).
- A valid local repository path on your machine that you intend to use as the project codebase.

## Steps

1. **Access the Dashboard**: Open your web browser and navigate to `http://localhost:4444`.
2. **Navigate to Project Creation**: Click the **New Project** button located in the primary navigation menu.
3. **Configure Project Details**:
   - **Name**: Enter a clear, identifiable name for your project.
   - **Repository Path**: Enter the absolute file path to the local repository on your machine.
4. **Initialize Core Settings**: Review the default system settings and adjust them if necessary. Specifically, ensure the API Key is correctly inherited or overridden for this project.
5. **Confirm and Create**: Click the **Create Project** button to finalize the setup.

## Expected Result

Upon successful creation, the dashboard will redirect you to the new project's workspace. The project will appear in the active projects list, and you will be able to start creating sprints and dispatching tasks for this specific codebase.
